# E. UI/UX 设计规则

## 适用场景
修改页面 UI、动画、布局、交互时加载此规则。

## 设计参考
- Apple Human Interface（WWDC 设计语言）
- Linear.app
- Vercel Dashboard
- Arc Browser
- Notion AI

## 设计语言 — Apple Glass Morphism
- 毛玻璃效果：`bg-white/70 backdrop-blur-xl border border-white/40`
- 大面积留白，信息层级清晰
- 蓝白渐变主色调（`from-sky-400 to-violet-500`）
- 圆角卡片（`rounded-[20px]` 到 `rounded-[24px]`）
- 柔和阴影（`shadow-[0_8px_30px_rgba(0,0,0,0.04)]`）
- 克制动画，不使用夸张特效
- 所有元素保持一致的视觉语言

## 颜色系统
- 主色调：sky-400 → violet-500 渐变
- 辅助色：amber（提示）、emerald（成功）、rose（错误）
- 背景：`#f3f6fb`
- 文字：`#09111f`（主要）, `#63748c`（次要）
- 玻璃面板：`rgba(255,255,255,0.72)` 背景
- 边框：`rgba(255,255,255,0.58)`

## 页面布局
- 最大宽度：`max-w-5xl` 或 `max-w-6xl`
- 上下留白充足：`px-5 pb-24 pt-12`
- Hero 区域宽：`max-w-[560px]`
- 响应式：768px 以下切换为上下排列

## 交互原则
- **取消所有工作流概念**：不出现步骤条、下一步按钮、流程切换
- **3秒理解功能，10秒完成操作**：选择类型 → 粘贴 → 点击 → 出结果
- **不要全屏 Loading**：按钮内旋转动画即可
- **不要刷新页面**：结果区域平滑展开
- **所有数据展示优先使用**：卡片、列表、表格，避免复杂图表
- **模式切换**：整个页面同步变化（300ms 淡入淡出）

## 动画要求
- Hover：`scale(1.02)` + 阴影增强，300ms
- 点击：`scale(0.98)`，100ms
- 出现：`opacity + translateY`，300ms
- 数字：CountUp 动画，1200ms
- 背景光晕：缓慢漂浮，不闪烁
- 支持 `prefers-reduced-motion`

## 重要页面结构
### AI优化页面
Hero → 两个大卡片选择 → 输入区 → 优化按钮 → 结果区域（评分+建议+对比+说明）

### 首页
导航栏 → Hero → 功能入口 → 统计数字

### 资料中心
搜索框 + 快捷入口（我的上传/收藏）→ 筛选栏 → 内容列表

## 禁止事项
- 禁止出现步骤条、进度条指示器
- 禁止全屏 Loading 遮罩
- 禁止上传/拖拽/文件入口（仅粘贴文字）
- 禁止廉价网页风格
- 禁止过度堆叠卡片（每屏不超过 4 张卡片）
- 禁止视觉元素之间出现多个不同风格的阴影
- 禁止复杂操作流程，所有功能必须在同一页面完成
